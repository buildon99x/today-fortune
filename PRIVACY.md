# 프라이버시 정책

마지막 갱신: 2026-05-29

## 한 줄

**서버는 데이터베이스를 사용하지 않습니다.** 모든 식별자는 HMAC-SHA256으로 해시 저장하고, 1시간 후 자동 폐기됩니다. 사용자 계정·로그인·요청 바디 로그 모두 없음.

## 서버가 받는 것 / 받지 않는 것

받는 것:
- 사주 입력: 생년월일·시·성별·음양력
- 통신 IP (TCP 연결 시 자동)
- IAP/광고 영수증 (프리미엄 해제 시)

받지 않는 것:
- 이름, 이메일, 전화번호, 주소
- 계정 ID, 로그인 토큰
- 디바이스 ID, 광고 ID(IDFA/AAID)

## 서버에 머무는 것

| 항목 | 어떤 형태로 | 어디 | 얼마나 |
|---|---|---|---|
| 생년월일 → 운세 캐시 | **HMAC-SHA256 해시 키**, 응답 본문은 값 | Upstash Redis 또는 인메모리 | **기본 비활성(0초)**. env로 활성화 시 명시 TTL |
| IP → 레이트리밋 카운터 | **HMAC-SHA256 해시 키**, 카운트만 | 같음 | 60초 윈도우 |
| 영수증 ID → 리플레이 방어 | **HMAC-SHA256 해시 키**, set membership만 | 같음 | 30일 (영수증 보존 권장 기간) |

### 알려진 사이드채널 — 캐시 활성 시

캐시를 활성화(`FORTUNE_CACHE_TTL_SEC > 0`)하면 응답시간 차이(LLM ~30s vs 캐시 ~0ms)로 "이 프로필이 이미 조회됐는가?"를 외부에서 추정할 수 있습니다. 가족·지인·유출 명단 대조 공격에 활용 가능. **기본 비활성으로 출고** — 비용 절감 목적의 활성화는 운영자가 명시적으로 결정하고 위 위험을 수용해야 합니다.

원본 평문은 **요청 처리 시점의 프로세스 메모리에만** 잠시 존재하고, 응답 후 GC 됩니다. 저장소에는 **언제나 해시만** 남습니다. Redis 덤프 유출 시에도 사용자 식별 불가.

## DB 없음 — 운영 부담 0

- 마이그레이션 없음
- 백업 정책 없음 (잃을 사용자 데이터 없음)
- 삭제 요청 대응 → 캐시 TTL(최대 1h) 자동 폐기. 즉시 삭제가 필요하면 `FORTUNE_CACHE_TTL_SEC=0` 설정으로 캐시 자체를 끔.

## 외부 전송

| 곳 | 무엇이 | 비고 |
|---|---|---|
| Anthropic API | 시스템 프롬프트 + 생년월일 포함 user 메시지 | Anthropic 보존 정책 적용. **ZDR 등 추가 계약 없이는 단축되지 않음** — 운영자가 별도 체결 필요 |
| Apple StoreKit2 | 영수증 JWS | 검증 응답만 사용, 저장 안 함 |
| Google Play | 영수증 + 서비스 계정 JWT | 검증 응답만 사용, 저장 안 함 |

## 운영자 설정 체크리스트

```bash
# 운영에선 반드시 — production에서 미설정 시 서버 부팅 거부.
PRIVACY_HMAC_SECRET=<openssl rand -hex 32>

# 캐시 정책 — 기본 0(휘발). 비용 절감 위해 활성화 시 캐시 타이밍 oracle 수용.
# FORTUNE_CACHE_TTL_SEC=0       # 기본, 권장
# FORTUNE_CACHE_TTL_SEC=3600    # 명시 활성화(1h)
```

## 기계가독 정책

`/api/privacy` 엔드포인트가 위 정책을 JSON으로 반환합니다. 토스 미니앱 심사·외부 감사 도구가 검증 가능.

```bash
curl https://<your-domain>/api/privacy
```

## 검증 가능한 사실

- 서버 코드 어디에도 `INSERT`/`CREATE TABLE`/`UPDATE` SQL 없음 (DB 자체 없음)
- 요청 바디는 console.log 대상 아님 (`server/src/app.mjs` 확인)
- 모든 식별자 저장 경로는 `hashIdentifier()` 통과 (`server/src/util/hash-id.mjs`)

수정·확인 PR은 환영합니다.
